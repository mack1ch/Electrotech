# ADR 0001: Монорепозиторий на pnpm и Turborepo

## Статус

Принято

## Контекст

Проекту нужна единая кодовая база для нескольких приложений и общих библиотек с предсказуемой локальной средой и хорошей поддержкой LLM-ассистирования.

## Решение

Использовать **монорепозиторий** с:

- **pnpm workspaces** для связи пакетов и детерминированных установок;
- **Turborepo** для кэшируемых и параллельных задач `dev`, `build`, `lint`, `typecheck`.

## Последствия

- Общие конфиги выносятся в `packages/config-*`.
- Скрипты в корне делегируют работу в воркспейсы через `turbo`.
- Новые приложения добавляются в `apps/` и в `pnpm-workspace.yaml` (уже покрыто glob).

## Альтернативы

- Несколько репозиториев — выше накладные расходы на синхронизацию версий общих пакетов.
- npm/yarn без Turborepo — слабее кэширование и оркестрация задач на масштабе монорепо.
