# ADR 0003: Правила LLM-ассистирования в репозитории

## Статус

Принято

## Контекст

Разработка ведётся с опорой на Cursor и LLM. Без явных правил модели склонны добавлять спекулятивные абстракции, доменные типы и несогласованную структуру.

## Решение

- Ввести **AGENTS.md** как обязательную точку входа для агентов.
- Держать **docs/** с архитектурой, инженерными правилами и playbook для LLM.
- Фиксировать крупные решения в **ADR**.
- Запретить внедрение доменной сложности и лишних технологий без явного скоупа задачи (см. AGENTS.md).

## Последствия

- Первый шаг при онборде человека или модели — чтение AGENTS.md и релевантных docs.
- Изменение процесса сотрудничества с LLM требует обновления AGENTS.md и при необходимости нового ADR.

## Альтернативы

- Полагаться только на code review — недостаточно для масштаба автоматических диффов.
- Не документировать — высокий риск архитектурного дрейфа.
