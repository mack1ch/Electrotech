# ADR 0002: Отдельные приложения web, supplier и api

## Статус

Принято

## Контекст

Продукт включает публичную зону и отдельную зону поставщика/администрирования, плюс серверный API. Смешивание всего в одном Next.js-приложении усложняет границы доступа, деплой и развитие.

## Решение

- **`apps/web`** — Next.js (App Router) для публичного лендинга и публичного сервиса.
- **`apps/supplier`** — отдельное Next.js-приложение для поставщиков и админских сценариев.
- **`apps/api`** — NestJS как единая HTTP-оболочка бэкенда.

Общие презентационные примитивы — в **`packages/ui`**.

## Последствия

- Два фронтенд-процесса и один API; переменные окружения и Docker-сервисы настроены раздельно.
- Дублирование некоторой инфраструктуры Next (минимально) принимается ради явных границ.

## Альтернативы

- Один Next.js с сегментацией маршрутов — меньше процессов, но слабее изоляция и выше риск смешения зон.
- BFF внутри Next — возможно позже точечно, не как замена `apps/api` без ADR.
