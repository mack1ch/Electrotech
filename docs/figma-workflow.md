# Работа с Figma через MCP

Для UI-задач **Figma считается источником правды**, когда макет или дизайн-система доступны в Figma.

## Принципы

1. **Не придумывать** визуальные решения, если для экрана есть утверждённый макет — реализация должна опираться на фрейм.
2. **Инспекция до кода**: через Figma MCP просмотреть нужный фрейм/компонент (иерархия, размеры, отступы, типографика, цвета, состояния).
3. **Токены и сетка**: выравнивание по сетке макета; при появлении дизайн-токенов в проекте — согласовывать именование с командой и отражать в `packages/ui` или стилях приложения осознанно.
4. **Неточности**: при противоречиях в макете, отсутствии состояния или размера — **зафиксировать вопрос** в задаче или комментарии, а не масштабно отклоняться от дизайна молча.

## Практический порядок

1. Определить файл/страницу и узел (frame/component) в Figma.
2. Снять структуру и ключевые размеры через MCP.
3. Реализовать экран минимальным набором компонентов; вынести повтор в `packages/ui` при втором осмысленном использовании (или по задаче).
4. Сверить ключевые отступы и размеры шрифтов с макетом.

## Связь с кодом

- Публичные экраны — преимущественно `apps/web`.
- Экраны поставщика/админки — `apps/supplier`.
- Переиспользуемые элементы без привязки к маршруту — `packages/ui`.
