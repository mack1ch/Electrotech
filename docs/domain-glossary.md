# Глоссарий (уровень проекта)

Краткие термины **без** детальной предметной модели. Расширение глоссария — по мере появления реальных доменных определений в задачах.

| Термин                     | Значение в контексте репозитория                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **Лендинг**                | Публичные страницы представления продукта; живут в зоне `apps/web` (или выделенных маршрутах по согласованию).               |
| **Публичный сервис**       | Пользовательская часть открытого сервиса; по умолчанию относится к публичному приложению (`apps/web`), пока не решено иначе. |
| **Зона поставщика**        | Приложение `apps/supplier`: кабинет поставщика и связанные внутренние сценарии.                                              |
| **Зона администрирования** | Часть функциональности, размещаемая в `apps/supplier`, если не появится отдельное решение в ADR.                             |
| **Приложение (app)**       | Отдельный деплойный юнит в `apps/*` с собственным `package.json` и жизненным циклом.                                         |
| **Пакет (package)**        | Переиспользуемый модуль в `packages/*`, подключаемый через pnpm workspace.                                                   |
| **Монорепозиторий**        | Один Git-репозиторий с несколькими приложениями и пакетами, общими инструментами и документацией.                            |
| **Общий UI**               | Пакет `packages/ui`: примитивы и мелкие компоненты для переиспользования в Next-приложениях.                                 |
| **Инфраструктура**         | PostgreSQL, Redis, Docker Compose и сопутствующие сервисы для локальной и будущей эксплуатации.                              |
