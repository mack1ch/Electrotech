# LLM playbook

Руководство для моделей и агентов, работающих с этим репозиторием в Cursor и схожих средах.

## Перед изменениями

1. Прочитать [AGENTS.md](../AGENTS.md).
2. Открыть релевантные разделы [architecture.md](architecture.md) и [engineering-rules.md](engineering-rules.md).
3. При затрагивании структуры репозитория — проверить [docs/adr/](adr/).

## Планирование

- Кратко определить цель задачи и **границы** диффа (какие приложения и пакеты затрагиваются).
- Не переписывать несвязанные файлы «заодно».

## Объём изменений

- Предпочитать **инкремент**: маленькие коммиты/PR с понятным описанием.
- Избегать **спекулятивной сложности**: никаких «каркасов на все случаи» без запроса.
- Сохранять **соглашения** репозитория: стиль импортов, именование, форматирование Prettier.

## Допущения

- Если требование неясно, **задокументировать допущение** в описании PR или в комментарии к задаче, а не внедрять догадку как факт.
- Не придумывать доменные сущности и API «от себя».

## После изменений

- Убедиться, что затронутые пакеты собираются и проходят lint/typecheck (или объяснить, почему шаг не применим).
- При архитектурном сдвиге — обновить документацию и ADR.

## Взаимодействие с дизайном

- Следовать [figma-workflow.md](figma-workflow.md): макет важнее импровизации.

## Краткий чеклист

- [ ] Прочитан AGENTS.md и нужные docs
- [ ] Дифф минимален и по задаче
- [ ] Нет лишних абстракций и доменного шума
- [ ] Документация согласована с изменениями (если нужно)
