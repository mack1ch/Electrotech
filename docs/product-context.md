# Контекст продукта (высокий уровень)

Проект **Электротехника** — это цифровой продукт с несколькими зонами ответственности. На этапе инициализации репозитория **детальная продуктовая логика намеренно не зафиксирована**.

## Планируемые зоны продукта

1. **Публичный лендинг** — представление продукта и входные точки для аудитории.
2. **Публичный сервис** — пользовательская часть сервиса в открытом доступе (конкретные сценарии будут определены позже).
3. **Зона поставщика и администрирования** — отдельное приложение для участников цепочки поставки и внутренних операций (без детализации ролей на данном этапе).

## Приоритеты развития инженерной базы

- Сохранять **чёткие границы** между приложениями и пакетами.
- Поддерживать **воспроизводимую локальную среду** (включая Docker).
- Документировать значимые решения в **ADR** и держать **AGENTS.md** актуальным для LLM-ассистирования.

## Что намеренно не описано здесь

- Каталоги, заказы, платежи, юридические сущности.
- Модель данных и интеграции.
- Детальные пользовательские роли и права.

Эти темы появляются в документации и коде **только после** явного продуктового и инженерного скоупа.
